Investigation plan

Dockhand Without Losing Declarations

Dockhand looks valuable as an admin and observability plane. The safe path is to keep AegisNode/Git as the owner of deployed state until Dockhand stack records, secrets, and mutations can be recreated deterministically.

Recommendation: hybrid pilot Source of truth: AegisNode config repo Open risk: socket permissions Open risk: Dockhand DB state

Short Answer

The idea is not silly. It becomes risky only if Dockhand and AegisNode both own stack lifecycle. Use Dockhand first for logs, container inspection, and operational visibility while AegisNode keeps stack import, route review, secrets, generated overrides, deployment, drift checks, and deletion cleanup.

Decision Frame

Success Criteria

  • Consumers still review and commit stacks/* before deploy.
  • Public exposure remains explicit through AegisNode metadata.
  • Dockhand cannot silently change AegisNode-owned stacks.
  • Logs replace Dozzle only if destructive actions stay blocked.
  • Beszel is removed only after host and container metric parity is proven.

Assumptions

  • AegisNode consumers value Git-backed reproducibility over one-click UI edits.
  • Current AegisNode route review is part of the product contract, not incidental UI.
  • Dockhand docs are accurate as of the manual version 2026-06-10.
  • No live Dockhand deployment has been tested against an AegisNode host yet.

Not A Goal Yet

  • No immediate removal of AegisNode stack management.
  • No hidden migration of AegisNode secrets into Dockhand's database.
  • No default Dockhand auto-update for AegisNode-managed containers.
  • No public Dockhand exposure without upstream auth and socket limits.

Current Ownership Boundary

AegisNode Today

  • Observability Compose is consumer-owned and Git-backed at stacks/observability/compose.yaml.
  • Deployment uses committed HEAD; dirty stacks/ content blocks deployment.
  • Application imports copy Compose into stacks/<name>/compose.yaml and write aegisnode.yaml.
  • Public services are separated from deployed services; AegisNode generates Pangolin labels in an override.
  • Runtime env values are profile secrets, written remotely as root-owned env files.
  • Existing socket proxy is read-oriented with POST: "0".

Dockhand Findings

Area What The Docs Say Impact On AegisNode
Stack source Dockhand is a centralized control plane; Compose files live on the Dockhand node for managed stacks. Conflicts with AegisNode's remote /opt/aegisnode/repository model unless Dockhand only observes or imports read-only.
Stack types Internal and adopted stacks are fully manageable; Git stacks sync from repos; untracked stacks are discovered with limited management. Untracked mode is the safest first state. Adoption or Git stack registration should be treated as a later migration.
REST API APIs can create stacks, update Compose content, deploy, and sync Git stacks. The API currently has no versioning. Automation is possible, but using it as AegisNode's backend adds dependency on Dockhand stack IDs and behavior.
Secrets Regular vars live in env files; secrets are encrypted in Dockhand DB and injected into the Compose process. Do not move AegisNode secrets into Dockhand DB unless backup, restore, restart, and rehydration semantics are designed.
Socket access Docs recommend a socket proxy for production, but warn Dockhand access remains powerful even through a proxy. Separate read-only observability from full operator mode. Free edition auth alone is not enough for multi-consumer admin isolation.
Labels dockhand.hidden, dockhand.update=false, dockhand.notify=false, URL, and ordering labels can steer Dockhand UI behavior. Useful for hiding plumbing and preventing auto-update of proxy/control-plane containers without making Dockhand authoritative.
Path mapping Dockhand recommends matching host/container data paths for stacks with relative bind mounts. Another reason not to let Dockhand redeploy existing AegisNode stacks until path handling is tested.

Replacement Readiness

Target Likely Outcome Gate Before Removal
Dozzle Good first replacement candidate. Dockhand logs include streaming, stopped container logs, ANSI rendering, pause/resume, download, and font sizing. Prove Dockhand can show logs through a constrained socket proxy without enabling terminal, delete, shell, or broad container mutation.
Beszel Maybe. Dockhand has stats and host info, but Beszel is purpose-built for lightweight system monitoring. Compare host CPU, memory, disk, container metrics, history retention, alerting, and multi-node story against Beszel on a real VPS.
Stack manager Do not replace yet. Dockhand's value is UI and operations, but AegisNode's value is declarative review, route contracts, and drift safety. Show that a fresh Dockhand DB can be recreated from the AegisNode config repo with no manual steps and no divergent secrets.

Consumer Confusion Risks

Duplicate Sources

Consumers may edit a Dockhand stack and expect Git to match. Mitigation: do not adopt AegisNode stacks during the pilot.

Visible Plumbing

Pangolin, Newt, socket proxy, and observability internals can clutter Dockhand. Mitigation: apply dockhand.hidden=true selectively.

Destructive UI

Free edition users are full admins. Mitigation: gate with Pangolin SSO and socket proxy permissions, or require Enterprise RBAC.

Auto-Update Drift

Dockhand can update containers outside Git. Mitigation: label AegisNode-managed containers with dockhand.update=false.

Secret Split

Dockhand DB secrets differ from AegisNode profile secrets. Mitigation: keep Dockhand out of app secret ownership at first.

Pilot Plan

Lab Install

Deploy Dockhand beside the current observability stack on a disposable host. Use Pangolin SSO, no public host port, and a dedicated socket proxy.

Verify: Dockhand sees AegisNode projects and basic logs without needing full Docker mutation rights.

Read-Only Boundary

Test a constrained socket proxy profile first. If Dockhand cannot function, document the minimum extra endpoints and the resulting risk.

Verify: no stop/delete/exec path works for ordinary access.

Dozzle Swap

Add Dockhand as an optional observability service and route dockhand.<domain>. Keep Dozzle until log parity is accepted.

Verify: common debugging workflows are better or at least equal.

Beszel Decision

Measure Dockhand's host and container metric retention against Beszel on the same VPS. Remove Beszel only if the loss is acceptable.

Verify: no observability gap for small VPS operations.

Implementation Shape If The Pilot Passes

AegisNode Changes

  • Add an optional Dockhand service to the generated observability Compose scaffold.
  • Expose dockhand.<domain> through Pangolin SSO and health checks.
  • Add generated labels for dockhand.update=false, dockhand.notify=false, and optional dockhand.hidden=true on infrastructure containers.
  • Keep stack add, stack env, generated Pangolin overrides, and Git drift checks unchanged.
  • Add tests proving Dockhand labels do not alter consumer-owned Compose files unless intentionally generated in overrides.

Boundaries To Preserve

  • AegisNode remains the only component that removes deleted Git stacks and Pangolin resources.
  • Dockhand cannot be required for provisioning, bootstrap, hardening, networking, proxy, or stack deploy.
  • AegisNode secrets stay in profile storage and root-owned remote env files, not Dockhand DB.
  • Dockhand API use is allowed only for read/query or explicitly documented optional integration.
  • Failure of Dockhand must not block AegisNode from deploying stacks.

Hard Gates Before Dockhand Owns Anything

Gate Required Proof Failing Result
Declarative reconstruction Empty Dockhand DB can be recreated from AegisNode repo/profile state. Dockhand remains observability only.
Secret ownership No app secret is only present in Dockhand DB, or DB backup/restore is a documented first-class contract. Dockhand cannot deploy AegisNode app stacks.
Socket scope Minimum socket proxy endpoints are listed and tested with destructive flows blocked or intentionally accepted. Keep Dozzle/Beszel or require Enterprise RBAC plus admin-only exposure.
Route model AegisNode route review and Pangolin resource IDs remain authoritative. Dockhand-created public exposure is out of scope.
Consumer UX A user can tell which UI is diagnostic and which command changes declared state. Do not surface Dockhand stack edit/deploy actions to consumers.
