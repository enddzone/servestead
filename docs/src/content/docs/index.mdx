Servestead turns a fresh Ubuntu VPS into a hardened, Git-backed host for private application stacks. The browser interface is the recommended way to work: it keeps profile values, repository state, run progress, diagnostics, and guarded cloud actions in context.

<ol class="workflow-strip" aria-label="Servestead workflow">
  <li class="workflow-step"><span>1</span><strong>Connect</strong><small>Use an existing Ubuntu VPS or create a DigitalOcean profile.</small></li>
  <li class="workflow-step"><span>2</span><strong>Review</strong><small>Confirm the profile, repository, and complete setup plan.</small></li>
  <li class="workflow-step"><span>3</span><strong>Run</strong><small>Watch each hardening and platform task complete in context.</small></li>
  <li class="workflow-step"><span>4</span><strong>Operate</strong><small>Manage stacks, GitOps state, access, cloud controls, and recovery.</small></li>
</ol>

:::caution[Begin with a disposable VPS]
Servestead changes SSH access, firewall policy, packages, Docker networking, reverse-proxy configuration, and cloud resources. Learn the workflow on a fresh server before using it on a machine that holds important data.
:::

Prefer the terminal? The [Terminal UI](reference/terminal-ui/) remains available for interactive DigitalOcean catalog selection, and the [CLI reference](reference/commands/) covers repeatable automation.
