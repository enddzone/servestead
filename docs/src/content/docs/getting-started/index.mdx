## The Browser Workflow

1. Open **Setup** and choose **Connect existing VPS**, or resume a saved profile.
2. Enter the profile name, server address, SSH key path, base domain, and certificate email.
3. Choose whether Servestead should create a local repository, use an existing checkout, or clone GitHub.
4. Read **Review and confirm**. Repository preparation and SSH execution have not started yet.
5. Select **Run after review** and follow the live status and logs.
6. Open **Profiles** after completion to verify setup, GitOps, recent runs, and diagnostics.

### Expected Result

The active profile reports a healthy or completed setup state, the configuration repository has a clear Git status, and the Command Center names the next useful action.

:::tip[Simple by default, details when needed]
The primary screens lead with one status and one next action. Expand profile metadata, repository actions, diffs, advanced setup fields, or diagnostics only when the task requires them.
:::

## Terminal Alternatives

Use `servestead setup` for the full-screen Terminal UI, including guided DigitalOcean catalog selection. Use `servestead setup --ip <address>` for a direct interactive run, or add `--yes` only when every required value is supplied for automation. See [Terminal UI](../reference/terminal-ui/) and [CLI commands](../reference/commands/).
