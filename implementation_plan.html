Blueprint Overview

VPS Automation & Zero-Trust Tunneling Pipeline

This architectural playbook maps out the end-to-end strategy to transform a clean, public Virtual Private Server (VPS) into an enterprise-grade hardened application stack. The system deploys a reverse-proxy routing engine, solves critical Docker-UFW security loopholes, and shuts down standard SSH exposure in favor of a user-space WireGuard Identity-Aware Tunnel (Pangolin).

Interactive Architecture Schema

1

Host Hardening Pipeline

Automates baseline Ubuntu OS security configurations, kernel hardening parameters (via sysctl), and continuous automated vulnerability upgrades.

2

Docker Network Encapsulation

Modifies the central Docker daemon configuration to completely disable raw iptables hijacking. Automatically creates explicit UFW routing paths for outgoing traffic.

3

Reverse Proxy & TLS Automation

Deploys Traefik configured with automated Let's Encrypt validation pathways and pairs it with the Pangolin identity-aware reverse proxy controller.

4

Zero-Trust Network Lockdown

Secures host SSH access, restricts public-facing ports, and maps all administrative capabilities directly onto multi-factor authenticated tunnels.

Native Task Runner (`tasks.go`)

package main

import (
	"context"
	"fmt"
	"io"
)

type Task struct {
	Name  string
	Apply string
}

func runTasks(ctx context.Context, client remoteClient, sshUser string, tasks []Task, progress io.Writer) error {
	for _, task := range tasks {
		if progress != nil {
			fmt.Fprintf(progress, "- %s\n", task.Name)
		}
		if err := client.Run(ctx, privilegedCommand(sshUser, task.Apply)); err != nil {
			return fmt.Errorf("task %q failed: %w", task.Name, err)
		}
	}
	return nil
}

func bootstrapTasks(config bootstrapConfig, adminPublicKey string) []Task {
	sshDirectory := "/home/" + config.AdminUser + "/.ssh"
	authorizedKeysPath := sshDirectory + "/authorized_keys"
	return []Task{
		{Name: "Install bootstrap packages", Apply: commandScript(
			aptInstallCommand("curl", "git", "gnupg2", "sudo"),
		)},
		{Name: "Create administrative group and user", Apply: commandScript(
			"getent group "+shellQuote(config.AdminUser)+" >/dev/null || groupadd "+shellQuote(config.AdminUser),
			"id -u "+shellQuote(config.AdminUser)+" >/dev/null 2>&1 || useradd --create-home --shell /bin/bash --gid "+shellQuote(config.AdminUser)+" --groups sudo "+shellQuote(config.AdminUser),
			"usermod --append --groups sudo "+shellQuote(config.AdminUser),
			"passwd -l "+shellQuote(config.AdminUser)+" >/dev/null 2>&1 || true",
		)},
		{Name: "Configure passwordless sudo", Apply: sudoersCommand(config.AdminUser)},
		{Name: "Create administrative SSH directory", Apply: commandScript(
			"install -d -m 0700 -o " + shellQuote(config.AdminUser) + " -g " + shellQuote(config.AdminUser) + " " + shellQuote(sshDirectory),
		)},
		{Name: "Install administrative public key", Apply: remoteWriteFileCommand(authorizedKeysPath, adminPublicKey+"\n", config.AdminUser, config.AdminUser, 0600)},
	}

PHASE 3 (40% to 60%)

Resolving Docker & UFW Conflict

Est. Time: 45 Mins

By default, when a Docker container exposes ports, it automatically injects custom `iptables` routes that entirely bypass system-level UFW controls. We eliminate this vulnerability by disabling automatic iptables modifications inside the Docker engine, while manually crafting UFW outbound masquerading routes to maintain internet connectivity for running containers.

  },

# At the top of /etc/ufw/before.rules - Before filter configurations:
# START AegisNode UFW MASQUERADE TRANSLATIONS
*nat
:POSTROUTING ACCEPT [0:0]
# Translate default and customized internal Docker network pools securely to the primary physical interface
-A POSTROUTING -s 172.17.0.0/16 -o eth0 -j MASQUERADE
-A POSTROUTING -s 172.18.0.0/16 -o eth0 -j MASQUERADE
COMMIT
# END AegisNode UFW MASQUERADE TRANSLATIONS

Ansible UFW & Docker Configuration Tasks (`docker_ufw.yml`)

---
- name: Resolve Docker & UFW Conflict
  hosts: all
  become: yes

  tasks:
    - name: Ensure Docker GPG keyring configuration is present
      apt_key:
        url: https://download.docker.com/linux/ubuntu/gpg
        state: present

    - name: Inject official Docker stable release package channel
      apt_repository:
        repo: "deb [arch=amd64] https://download.docker.com/linux/ubuntu {{ ansible_distribution_release }} stable"
        state: present

    - name: Install official Docker package runtimes
      apt:
        name:
          - docker-ce
          - docker-ce-cli
          - containerd.io
          - docker-compose-plugin
        state: present
        update_cache: yes

    - name: Write secured custom Docker engine daemon parameters
      copy:
        src: files/docker-daemon.json
        dest: /etc/docker/daemon.json
        owner: root
        group: root
        mode: '0644'
      notify: Restart docker service

    - name: Enforce POSTROUTING NAT adjustments inside UFW
      blockinfile:
        path: /etc/ufw/before.rules
        insertbefore: BOF
        block: |
          # START AegisNode UFW MASQUERADE TRANSLATIONS
          *nat
          :POSTROUTING ACCEPT [0:0]
          -A POSTROUTING -s 172.17.0.0/16 -o eth0 -j MASQUERADE
          -A POSTROUTING -s 172.18.0.0/16 -o eth0 -j MASQUERADE
          COMMIT
          # END AegisNode UFW MASQUERADE TRANSLATIONS

    - name: Establish secure system incoming drop posture
      ufw:
        policy: deny
        direction: incoming

    - name: Enable system outgoing transport pipeline configurations
      ufw:
        policy: allow
        direction: outgoing

    - name: Permit standard physical Web SSL traffic
      ufw:
        rule: allow
        port: "{{ item }}"
        proto: tcp
      loop: [80, 443]

    - name: Ensure UFW is active and enabled at runtime
      ufw:
        state: enabled

  handlers:
    - name: Restart docker service
      service:
        name: docker
        state: restarted

SSH Elimination & Tunnel Lockdown

Est. Time: 60 Mins

This final, critical phase seals all remaining public SSH ports. Access is routed exclusively through a private, authenticated user-space WireGuard interface mapped to the host's loopback interface. If container stacks drop, emergency access pathways are maintained through provider serial recovery methods.

UFW Firewall Transition & Lockdown Execution Script

#!/usr/bin/env bash

set -euo pipefail

echo "Executing secure physical network interface isolation pipeline..."

# 1. Determine active public interfaces and active virtual tunnel interfaces
PUBLIC_INTERFACE=$(ip route show | grep default | awk '{print $5}' | head -n1)
TUNNEL_INTERFACE="wg0" # Default interface associated with user-space WireGuard processes

echo "Targeting Public Interface: ${PUBLIC_INTERFACE}"
echo "Targeting Secure Tunnel Interface: ${TUNNEL_INTERFACE}"

# 2. Allow incoming Wireguard Handshake traffic publicly (UDP Port 51820)
sudo ufw allow in on "${PUBLIC_INTERFACE}" to any port 51820 proto udp comment 'Pangolin Tunnel Entrance'

# 3. Explicitly ALLOW SSH terminal access coming from the Zero-Trust interface
sudo ufw allow in on "${TUNNEL_INTERFACE}" to any port 22 proto tcp comment 'AegisNode Hardened SSH Access'

# 4. Deny incoming SSH connection requests on standard physical network interfaces
sudo ufw deny in on "${PUBLIC_INTERFACE}" to any port 22 proto tcp comment 'Deny Public SSH'

# 5. Reload active system rules
sudo ufw reload

echo "Public-facing SSH is now blocked. Access is isolated behind your Zero-Trust interface."

Emergency Recovery Protocols (Lockout Mitigation)

If your WireGuard/Pangolin container container stops or becomes corrupted, you will be locked out of the system via public SSH. In this scenario, do not panic. Use your cloud provider's emergency recovery systems:

  1. Log in to your provider panel (e.g., Hetzner Cloud or DigitalOcean console).
  2. Open the secure web-based Serial VNC Console. This console connects directly to the hypervisor's virtual screen, bypassing SSH and UFW networks completely.
  3. Authenticate using your unprivileged administrative account username and emergency password.
  4. Run sudo ufw disable to immediately bypass firewall block chains and allow temporary public-facing SSH access on port 22 for recovery.
